# API定義とモックのセットアップ

このドキュメントは、API定義の管理方法と、開発用のモックサーバーのセットアップ方針を定義します。

## API定義

- **OpenAPI 3 (Swagger)**: APIの仕様はOpenAPI 3フォーマットで記述します。
- **Springdoc**: SpringBootとOpenAPIを統合するためのライブラリ `springdoc-openapi` を利用します。これにより、コード（コントローラーやDTO）からOpenAPI仕様書が自動生成され、`swagger-ui` での確認が可能になります。
- **管理方法**: 生成された `openapi.yaml` はリポジトリで管理し、バージョン間の差分を確認できるようにします。

## モックサーバー

- **目的**: フロントエンド開発チームや他のバックエンドユニットが、依存するAPIの実装を待たずに開発を進められるようにするため。
- **ツール**: **`Prism`** を利用します。PrismはOpenAPI仕様書から自動的にモックサーバーを起動できるツールです。
- **運用**:
    1. 各ユニットは `openapi.yaml` を提供します。
    2. 開発者は `Prism` をローカルで実行し、必要なAPIエンドポイントをモックして開発を進めます。
    3. `docker-compose.yml` に `Prism` をサービスとして追加し、簡単に起動できるように構成します。
