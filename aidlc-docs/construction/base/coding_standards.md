# コーディング規約と静的解析

このドキュメントは、プロジェクトのコード品質を維持するためのコーディング規約と静的解析ツールのセットアップを定義します。

## フォーマッター

- **ktlint**: Kotlinの公式スタイルガイドに準拠したフォーマッター兼リンターです。Gradleプラグインを導入し、CIで自動的にチェックとフォーマット修正が実行されるように設定します。

## 静的解析

- **Detekt**: Kotlin用の静的コード解析ツールです。コードの複雑度、潜在的なバグ、コードスメルなどを検出し、品質を維持します。これもGradleプラグインを導入し、CIでレポートを生成するように設定します。

## 設定方針

- `build.gradle.kts` に `ktlint` と `detekt` のGradleプラグインと設定を共通で定義し、全てのサブモジュールに適用します。
- VSCodeなどのIDEでは、`ktlint` の拡張機能を導入し、保存時に自動フォーマットが適用されるように設定することを推奨します。
