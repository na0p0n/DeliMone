# ドメインモデル設計計画

このドキュメントは、日々の稼働記録機能（Unit 2）のドメインモデル設計のステップを定義します。

- [x] `DailyLog` コンポーネントのドメインモデルを `daily_log.md` に記述する。これは、その日の稼働記録全体を管理する集約ルートとなります。
- [x] `Break` コンポーネントのドメインモデルを `break.md` に記述する。これは、休憩時間を記録します。
- [x] `Delivery` コンポーネントのドメインモデルを `delivery.md` に記述する。これは、個々の配達の売上を記録します。
- [x] `Expense` コンポーネントのドメインモデルを `expense.md` に記述する。これは、日々の経費を記録します。
- [x] `Service` コンポーネントのドメインモデルを `service.md` に記述する。これは、配達サービスの種類を定義します。
- [x] すべてのモデル定義が完了したら、レビューと承認を求める。
