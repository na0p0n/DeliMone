# ユーザーストーリー開発計画

## 概要
このドキュメントは、デリバリー配達員の収益管理・可視化アプリ開発のユーザーストーリー実装計画を定義します。

## 開発ステップ

### Epic 1: サービス設定と支払いサイクル定義
- [ ] **Story 1.1:** 新しいデリバリーサービスを追加・編集できる機能を実装する。主要サービス（UberEats、出前館、Wolt、RocketNow、Amazon Flex、menu、ピックゴー）はプリセットから選択可能とし、手動での詳細設定も可能にする。
- [ ] **Story 1.2:** サービスごとに支払いサイクルを詳細に設定できる機能を実装する。
    - [ ] カスタム週払い設定（集計期間、支払日指定）
    - [ ] 月2回 / 営業日ベース設定（複数締め日、固定日 or 営業日後指定）
    - [ ] 手動/随時設定
- [ ] **Story 1.3:** 日本の祝日を考慮した入金予定日の自動計算ロジックを実装する（祝日データはGoogle Calendar API等の外部APIから取得）。

### Epic 2: 日々の稼働と勤怠の記録
- [ ] **Story 2.1:** 「稼働スタート」ボタンによる時間計測開始機能を実装する。
- [ ] **Story 2.2:** 「休憩」「再開」ボタンによる実稼働時間の記録機能を実装する。
- [ ] **Story 2.3:** サービスごとの売上入力機能を実装する。
- [ ] **Story 2.4:** 入力画面上で「今日の売上合計（全サービス合算）」をリアルタイムに確認できる機能を実装する。

### Epic 3: キャッシュフローとカレンダー連携
- [ ] **Story 3.1:** ホーム画面などで直近の入金予定リストを表示する機能を実装する。
- [ ] **Story 3.2:** 算出された入金予定日をGoogleカレンダーやAppleカレンダーに直接登録するためのリンクを生成する機能を実装する。
- [ ] **Story 3.3:** 入金実績を管理（受取済への変更、入金日修正）する機能を実装する。

### Epic 4: 売上と効率の分析
- [ ] **Story 4.1:** 「月ごと」「週ごと」の売上推移と内訳グラフを表示する機能を実装する。
- [ ] **Story 4.2:** 稼働時間と売上に基づき、時給（日次・期間平均）を算出して表示する機能を実装する。

### Epic 5: 非機能要件・スマホ最適化（PWA）
- [ ] **Story 5.1:** Webアプリをスマートフォンのホーム画面に追加（インストール）できるようPWAに対応する。
- [ ] **Story 5.2:** ホーム画面から起動した際に、アドレスバー非表示の全画面モードで利用できるよう設定する。
- [ ] **Story 5.3:** ダークモードに対応したUIを実装する。
