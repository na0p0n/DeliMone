# Unit 3: キャッシュフロー管理

この作業ユニットは、配達員の収益とキャッシュフローを可視化し、管理するための機能に焦点を当てています。

## User Stories

### Story 3.1: 入金予定カレンダー
**As a** 配達員,
**I want to** アプリ内カレンダーに入金予定日と金額が表示される,
**so that** いつ、いくら入金されるのかを視覚的に把握できる。

#### Acceptance Criteria
- [ ] アプリ内にカレンダービューがある。
- [ ] 計算された入金予定日がカレンダー上にマークされる。
- [ ] カレンダーの日付を選択すると、その日に入金が予定されているサービスの合計金額が表示される。

### Story 3.2: 入金予定リスト
**As a** 配達員,
**I want to** 直近の入金予定リストを表示する,
**so that** 今後の収入をリスト形式で簡単に確認できる。

#### Acceptance Criteria
- [ ] ホーム画面などに、今後1ヶ月分の入金予定がリストで表示される。
- [ ] リストには「入金予定日」「サービス名」「金額」が含まれる。
- [ ] 過去の入金予定は表示されない。

### Story 3.3: 外部カレンダー連携
**As a** 配達員,
**I want to** 入金予定日をGoogleカレンダーやAppleカレンダーに登録できる,
**so that** 普段使っているカレンダーで一元管理できる。

#### Acceptance Criteria
- [ ] 各入金予定に、GoogleカレンダーとAppleカレンダーへの登録リンクまたはボタンがある。
- [ ] リンクをクリックすると、予定のタイトル（例：UberEats入金）、日付、金額が入力された状態でカレンダーアプリが開く。

### Story 3.4: 入金実績管理
**As a** 配達員,
**I want to** 入金実績を管理できる,
**so that** 予定通りに入金されたかを確認できる。

#### Acceptance Criteria
- [ ] 入金予定リストやカレンダーで、入金済みのステータスに変更できる。
- [ ] 入金済みになった予定は、見た目が変わる（例：色が薄くなる、打ち消し線が付く）。
- [ ] 実際に入金された日付が予定と異なった場合に、日付を修正できる。
