# Unit 2: 日々の稼働記録

この作業ユニットは、配達員の日々の業務活動を記録するための機能に焦点を当てています。

## User Stories

### Story 2.1: 稼働時間の計測
**As a** 配達員,
**I want to** 「稼働スタート」「終了」ボタンで時間計測を制御できる,
**so that** その日の総稼働時間を正確に記録できる。

#### Acceptance Criteria
- [ ] ホーム画面に「稼働スタート」ボタンがある。
- [ ] 「稼働スタート」を押すと、時間の計測が開始される。
- [ ] 稼働中は「終了」ボタンが表示される。
- [ ] 「終了」ボタンを押すと、その日の稼働が終了し、総稼働時間が記録される。

### Story 2.2: 休憩時間の記録
**As a** 配達員,
**I want to** 「休憩」「再開」ボタンで実稼働時間を記録できる,
**so that** 休憩時間を除いた純粋な稼働時間を把握できる。

#### Acceptance Criteria
- [ ] 稼働中に「休憩」ボタンが表示される。
- [ ] 「休憩」ボタンを押すと、休憩時間の計測が開始される。
- [ ] 休憩中は「再開」ボタンが表示される。
- [ ] 「再開」ボタンを押すと、稼働時間の計測が再開される。
- [ ] 総稼働時間から休憩時間が差し引かれ、実稼働時間が計算される。

### Story 2.3: 売上入力
**As a** 配達員,
**I want to** 1配達ごとの売上を入力できる,
**so that** 詳細な収益記録を作成できる。

#### Acceptance Criteria
- [ ] 配達完了ごとに売上金額を入力できるシンプルなフォームがある。
- [ ] どのサービスの配達だったかを選択できる。
- [ ] 入力された売上は日付と関連付けて保存される。

### Story 2.4: 経費記録
**As a** 配達員,
**I want to** 燃料費などの経費を記録できる,
**so that** 純利益を正確に把握できる。

#### Acceptance Criteria
- [ ] 経費の金額と内容（例：ガソリン代、備品費など）を入力できる。
- [ ] 経費は日付と関連付けて保存される。

### Story 2.5: 売上サマリー表示
**As a** 配達員,
**I want to** 「今日の売上合計」をリアルタイムに確認できる,
**so that** その日の目標達成度を常に把握できる。

#### Acceptance Criteria
- [ ] ホーム画面や入力画面に、その日に記録された全サービスの売上合計が常に表示される。
- [ ] 売上を入力するたびに、合計金額が即座に更新される。
