# Unit 1: サービス設定

この作業ユニットは、アプリケーションの基本的な設定機能、特にデリバリーサービスと支払いサイクルの管理に焦点を当てています。

## User Stories

### Story 1.1: サービス管理
**As a** 配達員,
**I want to** 新しいデリバリーサービスを追加・編集できる,
**so that** 自分が利用している全てのサービスの収益を管理できる。

#### Acceptance Criteria
- [ ] 主要なデリバリーサービス（UberEats, 出前館, Wolt, RocketNow, Amazon Flex, menu, ピックゴー）がプリセットとして提供されている。
- [ ] プリセットからサービスを簡単に追加できる。
- [ ] プリセットにないサービスも手動で名称などを設定して追加できる。
- [ ] 登録済みのサービス情報を編集できる。
- [ ] 登録済みのサービスを削除できる。

### Story 1.2: 支払いサイクル設定
**As a** 配達員,
**I want to** サービスごとに支払いサイクルを詳細に設定できる,
**so that** 正確な入金予定日を把握できる。

#### Acceptance Criteria
- [ ] サービスごとに支払いサイクルを設定できる画面がある。
- [ ] 週払いの設定ができる（例：毎週水曜日締め、翌週金曜日払い）。
- [ ] 月2回払いの設定ができる（例：15日締め25日払い、末日締め翌月10日払い）。
- [ ] 締め日や支払日が固定日か営業日かを指定できる。
- [ ] 手動での入金確認（随時払い）を設定できる。

### Story 1.3: 入金予定日の自動計算
**As a** 配達員,
**I want to** 日本の祝日を考慮した入金予定日が自動で計算される,
**so that** 支払日が祝日や週末と重なった場合の正確な入金日を知ることができる。

#### Acceptance Criteria
- [ ] 支払日が土日祝日と重なる場合、翌営業日に入金予定日が自動で調整される。
- [ ] 日本の祝日データは外部API（例：Google Calendar API）から取得し、正確性が担保される。
