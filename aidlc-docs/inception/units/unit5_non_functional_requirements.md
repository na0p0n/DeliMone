# Unit 5: 非機能要件

この作業ユニットは、アプリケーションの使いやすさやモバイルデバイスでの体験を向上させるための技術的な要件に焦点を当てています。

## User Stories

### Story 5.1: PWA対応
**As a** 配達員,
**I want to** Webアプリをスマートフォンのホーム画面に追加（インストール）できる,
**so that** ネイティブアプリのように手軽にアクセスしたい。

#### Acceptance Criteria
- [ ] WebアプリがPWA（Progressive Web App）の基準を満たしている。
- [ ] スマートフォンのブラウザから「ホーム画面に追加」のアクションを実行できる。
- [ ] ホーム画面に追加されたアイコンからアプリを起動できる。

### Story 5.2: 全画面モード
**As a** 配達員,
**I want to** ホーム画面から起動した際に、アドレスバー非表示の全画面モードで利用できる,
**so that** より広い画面でアプリに集中したい。

#### Acceptance Criteria
- [ ] PWAとしてインストールされたアプリを起動すると、ブラウザのアドレスバーなどが表示されないスタンドアロンモードで動作する。

### Story 5.3: ダークモード対応
**As a** 配達員,
**I want to** ダークモードに対応したUIを利用できる,
**so that** 夜間や暗い場所でも目の負担を少なくアプリを使いたい。

#### Acceptance Criteria
- [ ] アプリ内にライトモードとダークモードを切り替える設定がある。
- [ ] OSの表示モード設定（ライト／ダーク）を自動で検知し、アプリのテーマに反映する。
- [ ] ダークモードでもテキストやグラフの視認性が十分に確保されている。
